  NPV table:

  +------+--------+--------+

  | id   | year   | npv    |

  +------+--------+--------+

  | 1    | 2018   | 100    |

  | 7    | 2020   | 30     |

  | 13   | 2019   | 40     |

  | 1    | 2019   | 113    |

  | 2    | 2008   | 121    |

  | 3    | 2009   | 12     |

  | 11   | 2020   | 99     |

  | 7    | 2019   | 0      |

  +------+--------+--------+



  Queries table:

  +------+--------+

  | id   | year   |

  +------+--------+

  | 1    | 2019   |

  | 2    | 2008   |

  | 3    | 2009   |

  | 7    | 2018   |

  | 7    | 2019   |

  | 7    | 2020   |

  | 13   | 2019   |

  +------+--------+



  Result table:

  +------+--------+--------+

  | id   | year   | npv    |

  +------+--------+--------+

  | 1    | 2019   | 113    |

  | 2    | 2008   | 121    |

  | 3    | 2009   | 12     |

  | 7    | 2018   | 0      |

  | 7    | 2019   | 0      |

  | 7    | 2020   | 30     |

  | 13   | 2019   | 40     |

  +------+--------+--------+



  The npv value of (7, 2018) is not present in the NPV table, we consider it 0.

  The npv values of all other queries can be found in the NPV table.

  </pre>
